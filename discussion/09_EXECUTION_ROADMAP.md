## 二、执行计划：4周冲刺

### 原则

1. **先修正确性，再优化性能，最后加功能**
2. **每个 PR 聚焦单一问题，便于 review 和回滚**
3. **先跑通再优化——不追求完美，追求可用**
4. **数据问题优先于代码问题——数据断裂比代码风格更致命**

## 五、MVP v1.0 定义

**核心价值**：让新手玩家在选牌时获得有用的推荐

### 5.1 MVP 必须完成（阻塞发布）

- [ ] 6 个角色数据引用完整，无断裂
- [ ] 选牌推荐算法准确（流派权重差异化 + 协同度统一）
- [ ] 移动端基本可用（汉堡菜单 + 响应式布局）
- [ ] 暗色主题完整
- [ ] 核心算法有测试保护
- [ ] 首屏加载 < 3s

### 5.2 MVP 应该完成（影响体验）

- [ ] 卡牌升级描述覆盖率 > 50%
- [ ] 事件数据 > 30 条
- [ ] 7 个关键词定义补全
- [ ] Error Boundary 全局捕获
- [ ] 搜索防抖

### 5.3 MVP 可以延后（不阻塞发布）

- [ ] PWA 离线支持
- [ ] 国际化
- [ ] Web Worker
- [ ] 多设备同步
- [ ] 社区数据贡献
- [ ] E2E 测试
