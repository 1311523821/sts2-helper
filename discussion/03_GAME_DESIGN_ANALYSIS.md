## 1. 卡牌数据完整性分析

### 1.1 各角色卡牌数量统计

| 角色 | 总卡牌数 | 基础 | 普通 | 稀有 | 罕见 | 特殊 | 数据质量 |
|------|---------|------|------|------|------|------|---------|
| 铁甲战士 (Ironclad) | 80 | 3 | 22 | 38 | 17 | 0 | ⭐⭐⭐⭐ |
| 猎手 (Silent) | 55 | 4 | 17 | 20 | 14 | 0 | ⭐⭐⭐⭐ |
| 故障机器人 (Defect) | 94 | 4 | 24 | 47 | 19 | 0 | ⭐⭐ |
| 观察者 (Watcher) | 53 | 4 | 21 | 16 | 12 | 0 | ⭐⭐⭐⭐ |
| 储君 (Prince) | 66 | 3 | 11 | 35 | 17 | 0 | ⭐⭐ |
| 亡灵法师 (Necromancer) | 79 | 4 | 17 | 33 | 23 | 2 | ⭐⭐⭐ |

### 1.2 严重问题：卡牌描述质量不一致

**问题1：大量卡牌缺少升级后描述**

几乎所有卡牌都缺少 `upgradedDescription` 字段：
- 铁甲战士：80张中仅3张有升级描述（基础牌）
- 猎手：55张中仅2张有
- 故障机器人：94张中仅2张有
- 观察者：53张中仅2张有
- 储君：66张中约8张有
- 亡灵法师：79张中约17张有

**影响**：玩家无法通过工具了解升级后的效果变化，这对升级决策至关重要。

**问题2：故障机器人和储君的卡牌描述包含主观评价**

这是**最严重的数据质量问题**。大量卡牌的 `description` 字段不是客观效果描述，而是社区玩家的主观评价：

```
defect_core_acceleration: "超模爆了，前期来上一张真的美滋滋。"
defect_shatter: "神中神，黑暗体系必带。"
defect_superluminal: "感觉这个比一代要拉了。"
defect_refraction: "三费太笨重了。"
defect_supercritical: "这东西我用的着实不敢用啊。"
defect_mod_rework: "看了之后不昏睡，过去的都是神人了。"
prince_flow_light: "两费跌15，垃圾。"
prince_charge: "神中神，喜欢塞状态牌是吧？"
```

**影响**：
- 新玩家看到"垃圾"、"神中神"等描述会困惑
- 主观评价会随版本更新而变化，维护成本高
- 降低了工具的专业性和可信度
- `description` 字段应该是客观的卡牌效果文本

**问题3：储君和亡灵法师的数据来源标记**

这两个角色标注了数据来源为"游侠网"、"游民星空"，说明数据是从第三方网站采集而非官方数据，可能存在翻译或数值错误。

### 2.2 严重问题：流派引用了大量不存在的卡牌

**这是整个项目最严重的数据完整性问题。** 经过交叉验证，以下流派的 coreCards、importantCards、supportCards 引用了大量在 `cards/*.json` 中不存在的卡牌ID：

**铁甲战士：**
- `ironclad_exhaust`（消耗流）：`ironclad_burning_pact`、`ironclad_dark_pact` 不存在
- `ironclad_block`（防战流）：`ironclad_metallicize_power` 不存在（正确ID应为 `ironclad_metallicize`）
- `ironclad_infinite`（无限流）：`ironclad_burning_pact` 不存在

**猎手：**
- `silent_poison`（毒流）：`silent_poison_cloud`、`silent_catalyst_power`、`silent_bouncing_flask` 不存在
- `silent_discard`（弃牌流）：`silent_sneaky_strike`、`silent_expertise`、`silent_bullet_time` 不存在

**观察者：**
- `watcher_divinity`（神格流）：`watcher_prostrate`、`watcher_simulate_dao`、`watcher_conclude`、`watcher_judgement` 不存在
- `watcher_retain`（保留流）：`watcher_well_laid_plans`（应为 `silent_well_laid_plans`？）、`watcher_just_lucky` 不存在
- `watcher_stance`（姿态流）：`watcher_indignation` 不存在

**储君：**（全部流派的核心卡都不存在！）
- `prince_succession`、`prince_crown_slash`、`prince_kingmaker`、`prince_throne_room`、`prince_royal_guard`、`prince_royal_decree`、`prince_tax`、`prince_knights_charge` —— **这些卡牌ID在 cards/prince.json 中完全不存在**

**亡灵法师：**（全部流派的核心卡都不存在！）
- `necromancer_raise_dead`、`necromancer_undead_legion`、`necromancer_bone_armor`、`necromancer_bone_storm`、`necromancer_soul_drain`、`necromancer_dark_pact`、`necromancer_graveyard_shift`、`necromancer_death_coil`、`necromancer_phylactery` —— **全部不存在**

**故障机器人：**
- `defect_dark`（黑暗球流）：`defect_darkness`、`defect_recursion` 不存在
- `defect_focus`（专注流）：`defect_core_memory` 不存在
- `defect_frost`（冰球流）：`defect_auto_shields` 不存在

**影响**：
- **流派匹配算法完全失效**：`matchArchetype()` 函数会因为找不到卡牌而返回错误结果
- **Combo检测完全失效**：大部分Combo引用的卡牌不存在
- **选牌推荐不可靠**：基于流派匹配的评分会产生错误
- **用户体验灾难**：工具推荐的"核心卡"实际上不存在于游戏中

### 2.3 流派定义的其他问题

**问题1：部分流派的卡牌权重过于统一**

大部分流派的核心卡权重分布为 `[90, 85, 80]` 或 `[95, 85, 80]`，缺少差异化。例如：
- 力量流的核心卡：燃烧(90)、恶魔形态(85)、愤怒(70) —— 这个分布是合理的
- 但消耗流的核心卡：腐败(95)、无惧疼痛(85)、恶魔之火(75) —— 恶魔之火的权重可能偏低

**问题2：流派之间的重叠度不足**

铁甲战士的力量流和放血流有大量重叠卡牌（如燃烧、恶魔形态），但流派定义中没有体现这种交叉关系。

**问题3：缺少"万金油"流派定义**

新手玩家经常需要的是"不追求特定流派，只拿好卡"的通用策略，但当前系统只有特定流派定义。

## 3. Combo系统分析

### 3.1 Combo数据完整性

全局 `combos/all.json` 包含19个Combo组合。经验证：

**引用不存在卡牌的Combo：**
- `silent_discard_cycle`：`silent_sneaky_strike` 不存在
- `defect_dark_recursion`：`defect_darkness`、`defect_recursion` 不存在
- `watcher_divinity_burst`：`watcher_conclude`、`watcher_judgement` 不存在
- `watcher_retain_shield`：`watcher_well_laid_plans` 不存在
- `necromancer_undead_army`：`necromancer_undead_legion`、`necromancer_bone_storm` 不存在
- `necromancer_contract_cycle`：`necromancer_dark_pact`、`necromancer_graveyard_shift` 不存在
- `prince_king_power`：`prince_succession`、`prince_kingmaker`、`prince_crown_slash` 不存在
- `prince_attack_burst`：`prince_kingmaker`、`prince_crown_slash`、`prince_throne_room` 不存在

**19个Combo中有8个（42%）引用了不存在的卡牌！**

### 3.2 Combo覆盖度分析

**缺少的跨流派Combo：**
- 铁甲战士：缺少"腐败+黑暗之拥+恶魔之火"的消耗流终极Combo
- 猎手：缺少"毒雾+催化剂+涂毒"的三重毒爆Combo（当前只有两重）
- 故障机器人：缺少"回声形态+电动力学"的电球终极Combo
- 观察者：缺少"急攻+发怒+连击风暴"的无限循环Combo的更详细描述

**缺少的跨角色Combo：**
当前系统不支持跨角色Combo（虽然杀戮尖塔2可能有多人模式），但可以考虑标记"理论协同"。

### 3.3 Combo强度评估

当前Combo的 `power` 字段分布：
- `game_winning`：14个（74%）
- `high`：5个（26%）

**问题**：`game_winning` 比例过高。实际上大部分Combo需要特定条件才能实现"game_winning"效果，应该更保守地标记强度。

## 4. 评分算法游戏性分析

### 4.1 选牌评分权重分析

`cardScorer.ts` 的评分维度权重：

| 维度 | 权重 | 评价 |
|------|------|------|
| 基础强度 (baseStrength) | 20% | ✅ 合理 |
| 流派适配 (archetypeFit) | 25% | ⚠️ 依赖流派数据准确性 |
| 牌库协同 (synergy) | 20% | ⚠️ 算法过于简单 |
| 楼层适配 (floorAdaptation) | 15% | ✅ 合理 |
| 遗物协同 (relicSynergy) | 10% | ⚠️ 遗物数据不完整 |
| 牌库健康度 (deckHealth) | 10% | ✅ 合理 |

### 4.2 算法问题分析

**问题1：流派适配权重过高且依赖错误数据**

`archetypeFit` 占25%权重，但如前所述，流派数据中大量卡牌引用不存在。这意味着：
- 如果一张卡在流派定义中被标记为核心卡（但ID不存在），它永远不会被匹配到
- 如果一张实际存在的好卡没有被任何流派引用，它的流派适配分为0

**问题2：协同度算法过于简单**

```typescript
function calcCardSynergy(cardId: string, deck: DeckCard[]): number {
  const card = getCardById(cardId)
  if (!card) return 0
  let synergy = 0
  for (const dc of deck) {
    const other = getCardById(dc.cardId)
    if (!other) continue
    const common = card.tags.filter(t => other.tags.includes(t))
    synergy += common.length * 12
  }
  return Math.min(100, synergy)
}
```

这个算法只检查 `tags` 的交集，存在以下问题：
- 没有考虑关键词（keywords）的协同，如"力量"+"多段攻击"
- 没有考虑卡牌之间的直接Combo关系
- 每个共同tag加12分，阈值设置过于武断
- 牌库越大，协同分越高（因为遍历的牌越多），这不一定正确

**问题3：楼层适配的逻辑过于简单**

```typescript
if (floor < 10) {
  if (card.type === 'attack') score += 15
  if (card.cost <= 1) score += 10
}
```

实际情况更复杂：
- 前期也需要防御牌（特别是面对精英时）
- 0费牌在后期不一定好（可能浪费手牌位置）
- 能力牌在前期也很有价值（如铁甲战士的燃烧）

**问题4：遗物协同检测不够精细**

```typescript
// 标签匹配
const commonTags = card.tags.filter(t => relic.tags.includes(t))
synergy += commonTags.length * 20

// 特殊协同检测
if (relic.tags.includes('力量') && card.tags.includes('力量')) synergy += 15
```

问题：
- 标签匹配和特殊检测有重叠计算
- 没有检测真正的遗物-卡牌协同（如"死灵之枝"+"消耗牌"）
- 遗物数据本身不完整，影响协同判断

### 4.3 跳过分析的合理性

`evaluateSkipOption` 函数的跳过逻辑：
- 牌库<15张时 +15分跳过值 → ❌ 前期应该多拿牌
- 牌库>30张时 +25分跳过值 → ✅ 合理
- 流派匹配>40且最佳选项<40时 +20分 → ⚠️ 依赖流派数据准确性
- 前期(floor<10) -15分 → ✅ 鼓励前期拿牌
- BOSS奖励(floor%16===0) -10分 → ✅ 合理

### 4.4 评分结果的游戏性问题

**新手体验问题：**
1. 推荐理由使用emoji但解释不够详细
2. 没有解释为什么某张牌比另一张好
3. 缺少"风险提示"（如"这张牌需要特定Combo才能发挥价值"）
4. 没有考虑当前生命值、金币等游戏状态

### 5.2 遗物数据问题

**问题1：缺少 boss/shop/event 类型遗物**

`relics/index.ts` 引入了 `boss.json`、`shop.json`、`event.json`，但这些文件不存在：
```typescript
import bossRelics from './boss.json'   // ENOENT
import shopRelics from './shop.json'   // ENOENT
import eventRelics from './event.json' // ENOENT
```

这会导致遗物池不完整，影响遗物推荐的准确性。

**问题2：初始遗物数据重复**

`starter.json` 和 `ironclad.json` 都包含了"燃烧之血"（burning_blood），`starter.json` 的ID是 `burning_blood`，而 `ironclad.json` 的ID是 `ironclad_burning_blood`。这会导致：
- 遗物映射中存在两个功能相同的遗物
- 算法可能重复计算协同效果

**问题3：遗物触发机制描述不够精确**

大部分遗物的 `effects` 只有一个 `trigger` 和简单的 `description`，缺少：
- 触发条件的精确数值（如"每打出10张攻击牌"中的10）
- 效果的持续时间
- 与其他遗物的交互规则

**问题4：新角色遗物数据可能不准确**

储君和亡灵法师的遗物数据来源标记为 `community`（社区），可能不如其他角色准确。

### 5.3 遗物与卡牌协同分析

当前的遗物协同检测机制：
```typescript
// 标签匹配
const commonTags = card.tags.filter(t => relic.tags.includes(t))
synergy += commonTags.length * 20
```

**问题**：协同检测过于表面化。真正的协同应该考虑：

| 遗物 | 协同卡牌 | 当前检测 | 实际协同 |
|------|---------|---------|---------|
| 死灵之枝 | 所有消耗牌 | ❌ 未检测 | 每消耗一张牌获得随机牌 |
| 卡戎之灰 | 所有消耗牌 | ❌ 未检测 | 每消耗一张牌AOE 3 |
| 永动陀螺 | 低费牌组合 | ❌ 未检测 | 手牌为空时抽牌 |
| 手里剑 | 多段攻击牌 | ⚠️ 部分 | 3攻击牌后力量+1 |
| 日晷 | 无限循环 | ❌ 未检测 | 洗牌2次后能量+2 |

|------|---------|---------|------|
| 第一章 | 8 | ⭐⭐⭐⭐ | ✅ 覆盖主要事件 |
| 第二章 | 3 | ⭐⭐ | ⚠️ 严重不足 |
| 第三章 | 1 | ⭐ | ❌ 几乎空白 |
| **总计** | **12** | | |

### 6.2 严重问题：事件覆盖度极低

杀戮尖塔2（Slay the Spire 2）预计有 **50+** 个事件，当前仅收录12个，覆盖率约24%。

**缺失的重要事件类型：**
- 第二章：大部分事件缺失（如"古老写作"、"被遗忘的祭坛"、"宝库"等）
- 第三章：几乎完全缺失（如"心灵祭坛"、"变异体"、"时间漩涡"等）
- 商店事件
- 多层选择事件

### 6.3 事件攻略质量

**当前事件数据结构：**
```json
{
  "id": "big_fish",
  "name": "大鱼",
  "options": [
    {
      "id": "big_fish_catch",
      "text": "捕捉它",
      "results": [
        { "type": "gold", "value": 15 },
        { "type": "damage", "value": 5 }
      ]
    }
  ]
}
```

**缺少的关键信息：**
1. **选择建议**：没有告诉玩家应该选哪个选项
2. **概率信息**：如"陵墓"事件有50%概率获得诅咒，但数据中没有体现
3. **条件触发**：某些选项需要特定条件（如金币、卡牌），没有标记
4. **角色差异**：不同角色的最佳选择可能不同

### 6.4 事件结果类型

当前支持的结果类型：
```
gold | heal | damage | upgrade | remove_card | transform_card | curse | potion | relic | card | combat | max_hp | heal_percent
```

**缺少的结果类型：**
- `max_hp_change`（最大生命值变化）
- `choice`（多选一，如选择获得哪种遗物）
- `conditional`（条件结果，如"如果你有X卡牌"）

## 7. Boss系统分析

### 7.1 Boss数据完整性

| 章节 | Boss数量 | 数据质量 | 评价 |
|------|---------|---------|------|
| 第一章 | 5 | ⭐⭐⭐⭐ | ✅ 完整（4精英+1章节Boss） |

| 第三章 | 3 | ⭐⭐⭐⭐ | ✅ 完整 |
| **总计** | **11** | | |

### 7.2 Boss攻略质量评估

**优点：**
- 每个Boss都有详细的技能列表
- 提供了针对性的策略建议
- 标记了弱点和威胁

**问题1：攻略缺乏流派针对性**

当前攻略是通用建议，没有针对不同流派给出具体建议。例如：
- 时间吞噬者对小刀流的影响应该特别强调
- 守卫者对毒流的优势应该具体说明

**问题2：缺少回合模式分析**

Boss的行为通常有固定模式（如拉加维林前3回合睡觉），但数据中没有体现这个模式。玩家需要知道的是"第几回合它会做什么"。

**问题3：缺少难度评级**

不同Boss对不同流派的难度差异很大，应该有更精细的难度评估。

### 7.3 Boss与选牌的关联

当前系统没有将Boss信息与选牌推荐关联。实际上：
- 第一章结束前应该考虑第一章Boss的弱点
- 面对时间吞噬者时应该避免小刀流
- 面对守卫者时毒流有优势

**建议**：在选牌评分中加入"Boss适配度"维度。

## 8. 游戏术语分析

### 8.1 关键词覆盖度

`keywords.ts` 定义了21个关键词，覆盖了主要游戏机制。

**已定义的关键词：**
- 状态类：易伤、虚弱、中毒
- 增益类：力量、敏捷、专注
- 机制类：护甲、消耗、虚无、保留、固有、抽牌
- 充能球类：生成球、球位
- 姿态类：愤怒、平静、神格
- 其他：召唤、伤口、眩晕、灼烧

### 8.2 缺失的关键词

卡牌中使用了以下关键词，但 `keywords.ts` 中没有定义：

| 关键词 | 使用场景 | 重要性 |
|--------|---------|--------|
| `scry` (预见) | 观察者多张卡牌 | ⭐⭐⭐⭐ 高 |
| `intangible` (无实体) | 故障机器人、猎手 | ⭐⭐⭐⭐ 高 |
| `star` (辉星) | 储君核心机制 | ⭐⭐⭐⭐⭐ 极高 |
| `evoke` (激活) | 故障机器人充能球 | ⭐⭐⭐⭐ 高 |
| `soul` (灵魂) | 亡灵法师核心机制 | ⭐⭐⭐⭐⭐ 极高 |
| `forge` (铸造) | 储君核心机制 | ⭐⭐⭐⭐⭐ 极高 |
| `doom` (灾厄) | 亡灵法师核心机制 | ⭐⭐⭐⭐⭐ 极高 |

**影响**：
- 新玩家无法通过工具了解这些核心机制
- 协同检测无法识别这些关键词的配合
- 降低了工具的教学价值

### 8.3 术语一致性问题

**问题1：同一概念使用不同术语**

- "护甲" vs "格挡"：`keywords.ts` 使用"护甲"，但卡牌描述中大量使用"格挡"
- "无实体" vs "无敌"：`keywords.ts` 没有定义"无实体"，但Boss攻略中使用"无敌"

**问题2：英文ID与中文名的映射**

部分关键词的英文ID不够直观：
- `channel` → 应为 `channel_orb` 更清晰
- `evoke` → 应为 `evoke_orb` 更清晰

## 9. 新手友好度分析

### 9.1 推荐理由质量

当前推荐理由示例：
```
"🗡️ 力量流的核心卡牌"
"⚔️ 力量流的重要卡牌"
"🤝 与牌库中的卡牌有良好协同"
"🗡️ 前期需要攻击牌提升输出"
"🔋 中后期能力牌价值高"
```

**问题：**
1. **过于笼统**：只说了"是核心卡"，没解释为什么
2. **缺少数值**：没有告诉玩家"这张牌能造成多少伤害"
3. **缺少上下文**：没有考虑当前牌库的具体情况
4. **缺少风险提示**：如"这张牌需要3回合setup"

### 9.2 新手学习路径

**当前缺失：**
1. **角色入门指南**：没有告诉新手"铁甲战士应该先学什么流派"
2. **卡牌优先级**：没有"前10回合应该拿什么牌"的指导
3. **常见错误**：流派guide中有 `commonMistakes`，但没有在选牌时展示
4. **术语解释**：新玩家看到"消耗"、"虚无"等术语可能不理解

### 9.3 新手体验改进建议

1. **添加"新手模式"**：简化推荐理由，只显示最重要的信息
2. **添加卡牌效果解释**：在推荐时解释关键词的含义
3. **添加流派入门提示**：如"这是力量流的第一张核心卡，拿到后可以开始走力量流"
4. **添加风险评估**：如"这张牌前期较弱，需要2-3回合setup"

```
├── 替换故障机器人/储君的主观描述
└── 修复遗物数据文件缺失（boss.json等）

高优先级（1-2周）：
├── 补充卡牌升级描述
├── 补充缺失的7个关键词定义

```

### 11.3 核心结论

**最严重的问题是数据引用断裂**：储君和亡灵法师的全部流派定义、以及故障机器人/观察者的部分流派定义，引用了在卡牌数据中完全不存在的卡牌ID。这导致：
- 流派匹配算法对这些角色完全失效
- Combo检测对这些角色完全失效
- 选牌推荐对这些角色不可靠

**建议**：在修复数据引用问题之前，应该在UI中明确标注"数据不完整"，避免误导玩家。
